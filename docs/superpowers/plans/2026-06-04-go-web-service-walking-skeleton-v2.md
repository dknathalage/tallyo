# Tallyo Web-Service Walking Skeleton v2 — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Wails desktop shell with a self-hosted Go web service (`tallyo serve`) that serves an embedded SvelteKit SPA, authenticates users (first-run setup + email/password sessions + manual invites), exposes one protected vertical slice (Business Profile) over a JSON API, and pushes live updates via SSE — proving the new architecture end-to-end.

**Architecture:** Single Go binary. chi router serves `/api/*` (JSON REST) and `/api/events` (SSE), and serves an embedded SvelteKit static build at `/` with `200.html` SPA fallback. Auth via `alexedwards/scs/v2` cookie sessions backed by a SQLite session table. Mutations flow handler → service → repository → audit → sqlc → modernc (tx commit), then the service broadcasts a change event to an in-process SSE hub. The reused Go data layer (`internal/db`, `internal/repository`, `internal/audit`) carries over; `sqlite.go` and `internal/service` are modified.

**Tech Stack:** Go 1.26, chi v5, alexedwards/scs/v2 (+ sqlite3store), golang.org/x/crypto/bcrypt, modernc.org/sqlite, sqlc, goose; SvelteKit + @sveltejs/adapter-static (Svelte 5 runes), Tailwind CSS 4, Vite.

**Spec:** `docs/superpowers/specs/2026-06-04-go-web-service-rewrite-design.md`

**Reuse note:** Follow the patterns in the existing `internal/repository/business_profile.go` (transactional audited upsert), `internal/audit/audit.go` (Execer interface), and `internal/db/migrations/00001_skeleton.sql` (goose) when writing new repositories/migrations/queries.

- [ ] **Step 3: Rewrite `Open` to apply PRAGMAs via the DSN (per-connection), then size the pool**

CRITICAL: `foreign_keys` and `busy_timeout` are **per-connection** pragmas — they reset on every new connection. The current `conn.Exec("PRAGMA …")` only configures ONE connection; with a pool of 8 the other 7 would run with `foreign_keys=OFF` and no busy timeout (breaking the `invites.created_by REFERENCES users(id)` FK and reintroducing `SQLITE_BUSY`). modernc applies DSN `_pragma` params on **every** connection — use that.

Replace the body of `Open` (the `sql.Open` + `SetMaxOpenConns(1)` + pragma loop, lines 33–47) with:
```go
	dsn := "file:" + path +
		"?_pragma=journal_mode(WAL)" +
		"&_pragma=foreign_keys(1)" +
		"&_pragma=busy_timeout(5000)"
	conn, err := sql.Open("sqlite", dsn)
	if err != nil {
		return nil, fmt.Errorf("open sqlite: %w", err)
	}
	// WAL permits concurrent readers with a single serialized writer; a small
	// pool suits a single-org self-hosted server. busy_timeout (per-connection,
	// via DSN above) makes writers wait rather than erroring SQLITE_BUSY.
	conn.SetMaxOpenConns(8)
	conn.SetMaxIdleConns(8)
	conn.SetConnMaxLifetime(0)
	return conn, nil
```
(The `os.MkdirAll` guard above it stays. The pragma `Exec` loop is removed entirely.)

package auth

import (
	"fmt"

	"golang.org/x/crypto/bcrypt"
)

// HashPassword returns a bcrypt hash. Rejects empty input.
func HashPassword(pw string) (string, error) {
	if pw == "" {
		return "", fmt.Errorf("password: empty")
	}
	b, err := bcrypt.GenerateFromPassword([]byte(pw), bcrypt.DefaultCost)
	if err != nil {
		return "", fmt.Errorf("hash password: %w", err)
	}
	return string(b), nil
}

// VerifyPassword reports whether pw matches the stored bcrypt hash.
func VerifyPassword(hash, pw string) bool {
	if hash == "" || pw == "" {
		return false
	}
	return bcrypt.CompareHashAndPassword([]byte(hash), []byte(pw)) == nil
}
```

package auth

import (
	"database/sql"
	"net/http"
	"time"

	"github.com/alexedwards/scs/sqlite3store"
	"github.com/alexedwards/scs/v2"
)

// NewSessionManager builds an scs manager backed by the sessions table.
// secure=true sets the Secure cookie flag (enable behind TLS).
func NewSessionManager(db *sql.DB, secure bool) *scs.SessionManager {
	m := scs.New()
	m.Store = sqlite3store.New(db) // starts a 5-min cleanup goroutine
	m.Lifetime = 7 * 24 * time.Hour
	m.Cookie.HttpOnly = true
	m.Cookie.SameSite = http.SameSiteLaxMode
	m.Cookie.Secure = secure
	m.Cookie.Path = "/"
	return m
}
```
Note for `cmd/tallyo`: the sqlite3store cleanup goroutine can be stopped via `sqlite3store.NewWithCleanupInterval`; for the skeleton the default is fine (process exits on shutdown).
